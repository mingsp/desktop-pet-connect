# Desktop Pet Connect 项目规划与任务记录

这份文档用于跟踪完整产品从原型到可给朋友试用版本的实现进度。它记录“要做什么、做到哪里、什么算完成”，避免产品方向、实现任务和已完成内容混在聊天里。

## 产品北极星

Desktop Pet Connect 是一款关系型 DIY 桌宠产品。用户可以把恋人、好友或重要的人做成桌宠，放在自己的电脑桌面上，并通过真实连接互相传消息、看状态、产生陪伴感。

核心判断：

- 我们不是做普通桌宠，也不是做普通聊天软件。
- 桌宠背后应该连接着一个真实的人。
- 情侣、好友、重要的人互相把对方做成桌宠，是产品最有方向性的场景。

## 当前阶段

当前处于 **Alpha 原型向可试用产品过渡阶段**。

目标不是继续做静态概念图，而是把已有 Electron 桌宠、宠物包导入、真实 Pet Connect 服务端和关系型产品方向整合成一个可给朋友试用的版本。

### M9 AI 增强

状态：后置规划

目标：

- AI 用来增强桌宠表达，不取代真实关系连接。

候选能力：

- 帮用户润色短消息。
- 根据关系称呼生成更自然的桌宠语气。
- 根据对方状态推荐轻量问候。
- 帮用户从头像/照片生成像素桌宠初稿。
- 使用 gpt-image2 生成商业级系统视觉资产，例如关系桌宠形象、品牌图、引导图和空状态图。
- 为情侣/好友生成专属动作或气泡文案。

原则：

- 默认关闭。
- 用户可控。
- 不自动替用户和真实朋友聊天。
- 不让 AI 抢走“真实人在连接”的产品核心。
- 生成图不能只停留在展示层，必须能被导入、校验、绑定和保存。

## 文档维护规则

- 每完成一个可验证功能，更新“当前任务看板”和“已完成记录”。
- 每新增一个重要设想，先进入对应设计文档，再决定是否进入任务看板。
- 不在路线图里记录具体代码细节，只记录业务价值、实现范围、状态和验收标准。
- 状态只使用：待办、进行中、部分完成、已完成、暂停。
- 优先级只使用：P0、P1、P2。
