# 桌宠产品蓝图

这份文档是产品蓝图总览，用来保持方向一致。具体功能设计拆到独立文档里维护，避免一个文件越来越乱。

## 产品愿景

做一个友好、有个性、可长期陪伴用户的关系型桌面伙伴。它不是单纯的动画播放器，而是一个轻量的桌面陪伴层：用户可以把恋人、朋友或重要的人做成桌宠，并通过这个形象完成情绪表达、互动和远程连接。

这个产品应该让普通用户很容易开始使用，也让创作者能持续带来新的角色和宠物包。

当前纠偏原则：默认体验不是小动物、不是复杂偏好中心，而是“把一个真实的人做成桌面陪伴，并和 TA 保持轻量连接”。高级配置可以存在，但不能成为朋友试用时的第一印象。

## 产品支柱

### 个人陪伴

- 可以给桌宠设置名字、性格、生日、备注和行为偏好。
- 可以把具体的人物形象做成桌宠，例如恋人、好友或重要的人。
- 可以选择、导入和切换宠物角色，不需要手动改文件。
- 日常控制保持克制：尺寸、透明度、置顶、穿透、任务栏显示、重置位置。
- 支持小型表达：问候、情绪、自动动作、天气播报。

### 创作者友好的宠物库

- 使用简单的宠物包格式：`pet.json` 加精灵图资源。
- 先支持文件夹导入，后续支持 `.petpack` 打包导入和导出。
- 导入前展示预览、名称、来源、动作支持情况和校验结果。
- 内置宠物和用户自定义宠物分开管理，避免产品更新覆盖用户内容。

详细设计见：[宠物库与宠物包设计](./pet-library.md)。

### 真实关系的桌面存在感

- 用户可以添加恋人或朋友的桌宠。
- 一对情侣可以互相把对方的形象做成桌宠，并让两个桌宠真实连接。
- 用户通过自己的桌宠给对方的桌宠发消息。
- 如果对方在线，对方桌宠实时收到消息，并用动作、气泡和通知表达出来。
- 这个体验应该像“角色来传话”，而不是又打开了一个完整聊天客户端。
- 关系边界必须是真实能力：用户可以隐藏在线、删除关系、屏蔽关系，并且这些状态要被服务端尊重。

详细设计见：[桌宠连接设计](./pet-connect.md)。

## 当前产品表面

Electron 原型已具备：

- 以 Friends Hub 为默认首页的关系入口：朋友列表、在线状态、桌面陪伴形象、最近来往、快捷传话和邀请入口。
- 与 V2 选中原型一致的暖色深色界面方向，减少设置中心感，优先呈现“一个重要的人在桌面上”的体验。
- 透明、无边框、置顶的桌宠窗口。
- 拖拽移动和小范围自动移动。
- 右键菜单和托盘恢复菜单。
- 重命名和名字显示。
- 尺寸、透明度、点击穿透、任务栏显示、重置位置。
- 档案、互动、天气、角色、行为等偏好设置页。
- 内置宠物和自定义宠物包加载。
- 角色切换、文件夹导入、自定义宠物目录打开。
- 宠物包导入前预览和基础校验。
- 悬停隐藏、点击反应、限制在屏幕内等行为控制。
- 根据城市播报天气。
- 基础全局快捷键。

当前产品缺口：

- 宠物包还缺少动作逐帧预览和 `.petpack` 打包导入/导出。
- 桌宠连接已有真实 WebSocket 链路，支持注册、邀请码加好友、在线状态、实时消息、离线消息、隐藏在线、删除关系和屏蔽关系；尚缺邀请链接、会话窗口和正式账号体系。
- 自动启动、更新、发布和创作者工作流还没有产品化。

## 近期优先级

1. 两台真实电脑 smoke test：安装、邀请码、在线状态、互发消息、隐藏在线、删除和屏蔽。
2. 关系主页首次启动引导：没有好友时一步步完成连接、邀请码和制作形象。
3. 增加邀请链接，降低朋友互相添加的成本。
4. 设计人物形象 DIY 流程，让用户更容易把恋人或朋友做成桌宠。
5. 增加宠物包动作逐帧预览和更细的错误定位。
6. 支持 `.petpack` 导入和导出。
7. 设计面向创作者的宠物包发布流程。
