# Friend Test Deployment

This is the alpha deployment checklist for giving the current desktop companion app to real friends and testing real invite-based messaging through the VPS service.

## Goal

Two Windows users should be able to:

- install or open the app on separate computers;
- connect to the shared VPS service;
- create and exchange real invite codes;
- see each other's online or away state;
- send short messages that appear through the desktop companion;
- keep local privacy controls available, including quiet mode and per-friend mute.

## Build The Windows App

Run from the project root:

```powershell
npm install
npm run build:portable
```

For a full installer plus portable build:

```powershell
npm run build:win
```

The generated files are written to `release/`. For a first friend test, the portable `.exe` is the easiest file to share because it does not require an installer flow.

## Default Server

The current desktop app defaults to:

```text
https://app.qimozhu.abrdns.com/pet-connect
```

That HTTPS endpoint is converted internally to a secure WebSocket connection for real-time presence and messages. No local server is required for normal friend testing.

## First Two-Person Test

1. User A opens the app and goes to Friends.
2. User A chooses Add by Invite and creates an invite code.
3. User B opens the app, chooses Add by Invite, and enters User A's invite code.
4. Both users confirm the other person appears in the friend list.
5. Both users check that status changes between Online, Away, and Quiet are reflected.
6. User A sends a short note from the Friends or Messages screen.
7. User B confirms the message appears in the app and through the desktop companion bubble.
8. User B replies, then both users close and reopen the app to confirm the relationship remains saved locally.
9. User A opens Privacy, hides online presence, and User B confirms User A appears offline.
10. User A tests Delete Relationship with a disposable test account and confirms messages no longer send.
11. User A tests Block with a disposable test account and confirms the blocked user cannot send messages or reconnect with a fresh invite.

## What To Send A Friend

- The portable `.exe` from `release/`, or the installer generated by `npm run build:win`.
- A short note that this is an unsigned alpha build, so Windows SmartScreen may show a warning.
- The expected server: `https://app.qimozhu.abrdns.com/pet-connect`.
- A fresh invite code created inside the app.

## Troubleshooting

If the app shows Disconnected:

- confirm the computer has internet access;
- confirm the server URL has not been changed in Settings;
- restart the app once;
- check the VPS service and reverse proxy if both users are disconnected.

If an invite code fails:

- create a new invite code because alpha codes can expire;
- remove extra spaces when pasting;
- confirm both users are connected to the same server endpoint.

If a friend is visible but messages do not appear:

- check Quiet Mode and per-friend mute;
- confirm incoming messages are allowed in Privacy;
- confirm the relationship was not deleted or blocked;
- ask the other user to keep the app running in the tray;
- try a short plain-text message under the character limit.

## Current Alpha Limits

- The app is not code-signed yet, so Windows may warn on first launch.
- Accounts are lightweight device identities, not full email or phone accounts.
- Relationship data and message history are stored locally on each computer.
- Delete and Block are available in Privacy, but account recovery and unblock management are not complete yet.
- Server backup and account recovery are not complete yet.

## Release Readiness Before Wider Sharing

- Add code signing or publish a clearly labeled unsigned alpha.
- Add unblock management and clearer account recovery.
- Add invitation links in addition to raw invite codes.
- Add VPS backup and restore notes.
- Run a two-machine smoke test after every release build.
