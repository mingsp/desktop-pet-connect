# 宠物库与宠物包设计

## 目标

宠物库负责让用户选择、导入、管理和未来分享桌宠角色。它应该让普通用户不碰文件也能完成角色切换，同时给创作者保留清晰的资源格式。

## 产品原则

- 内置宠物和自定义宠物分开管理，避免更新覆盖用户内容。
- 导入前必须先预览和校验，不能让坏包悄悄进入库。
- 宠物包格式保持简单，但运行时要允许更丰富的图集规格。
- 用户看到的是角色、动作和预览，不是工程文件结构。

## 当前能力

- 运行时从 `assets/pets/<pet-id>/` 读取内置宠物包。
- 自定义宠物导入到 Electron user-data 下的 `pets` 目录。
- 偏好设置里可以切换当前角色、导入角色、打开角色目录。
- 导入时会打开预览窗口，展示角色名、包 ID、精灵图、图集规格、动作状态、错误和警告。
- 校验通过后才允许“导入并切换”。

## 宠物包格式

最小结构：

```text
pet-folder/
├── pet.json
└── spritesheet.png
```

推荐结构：

```text
pet-folder/
├── pet.json
├── spritesheet.png
├── preview.png
└── frames/
```

`pet.json` 需要描述：

- `id`
- `displayName`
- `spritesheet`
- `grid`
- `cell`
- `states`
- `actions`

## 导入校验

当前基础校验：

- 是否存在有效的 `pet.json`。
- 精灵图路径是否存在，且必须是包目录内的相对路径。
- `grid.columns`、`grid.rows`、`cell.width`、`cell.height` 是否为正整数。
- 精灵图尺寸是否等于 `grid * cell`。
- `states` 是否存在。
- 每个状态的 `row` 是否越界。
- 每个状态的 `frames` 是否为正整数且不超过列数。
- `durations` 少于帧数时给警告。
- `actions` 指向不存在的状态时给警告。

下一步校验：

- 逐帧预览每个动作。
- 检查空白帧、边缘裁切、透明背景异常。
- 检查动作命名是否符合推荐语义。
- 检查预览图是否存在且比例合理。

## 后续方向

- 支持 `.petpack` 导入和导出。
- 支持删除、重命名和复制自定义宠物。
- 支持角色详情页：动作列表、作者、描述、版本、来源。
- 支持创作者发布流程和宠物包市场。
